## Why

Without cost visibility, agents become a roulette wheel. Phase D makes the price of every prompt visible in the title bar.

## Work

- Extend `AgentRegexParser`:
  - New patterns for token-usage detection
  - Claude Code typical formats (parsed from `--print` output and `/cost` slash command):
    - `(?i)(?:cache_creation_)?input_tokens?:\s*(\d+)` 
    - `(?i)output_tokens?:\s*(\d+)`
    - `(?i)cache(?:_creation|d)?_input_tokens?:\s*(\d+)`
  - Codex format: `(?i)total tokens:\s*(\d+)\s*\(input:\s*(\d+),\s*output:\s*(\d+)\)`
  - When a complete usage block parses, emit `.tokenUsage(input:, output:, cached:)`

- In `PaneViewModel.startObserving`, on `.tokenUsage` events:
  - Build a `CostEvent` using the current `model` (default to profile's hint) + PricingTable lookup
  - Append to `pendingCostEvents`
  - Schedule a Task to push to AppState.costLedger.record(event)

- AppState additions:
  - On init, spawn a Task subscribed to costLedger events that recomputes `todayCostUSD = costLedger.todayTotal()` and assigns it on @MainActor

- `CostPill` already takes `todayUSD:warnAtUSD:hardStopAtUSD:onTap:` ‚ÄĒ already bound in phase B's WindowOverlayBar; this phase just makes the value non-zero in practice

- Debug menu addition (DEBUG-only): `Hangar > Debug > Inject Fake Cost ($0.42)` ‚ÄĒ exercises the wiring without needing a live `claude` run

- Tests under `Tests/HangarCoreTests/Costs/`:
  - `TokenUsageParserTests` ‚ÄĒ feed canned chunks, assert correct CostEvent fields:
    - Claude: "input_tokens: 1234, output_tokens: 567" ‚Üí input=1234, output=567, cost via PricingTable
    - Codex: "total tokens: 1000 (input: 600, output: 400)" ‚Üí input=600, output=400
    - Mixed/incomplete chunks across multiple feeds
    - Pricing lookup uses currently-set model; falls back to profile.defaultModelHint

- Tests under `Tests/HangarKitTests/Pane/`:
  - `PaneViewModelCostForwardingTests` ‚ÄĒ when parser emits tokenUsage, PaneViewModel queues the cost event; verify the recorded event matches expected USD
